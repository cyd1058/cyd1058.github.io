Android 开发规范

1 介绍

好的开发团队应该是具有同样的语言、同样的理念、同样的规范去做同一件事。

1.1 目的

  • 减少维护花费。

  • 提高可读性。

  • 加快工作交接。

  • 减少名字增生。

  • 降低缺陷引入的机会。

1.2 术语和定义

  • 强制：编程时必须遵守的规定，含有强制字样或字体用加粗式样标注。

  • 推荐：编程时推荐遵守的规定，字体用普通式样标注。

2 文件组织

  • 避免超过 500 行的源文件。

  • 方法尽量不超过50行。

2.1 命名规范

  • 整体基则：统一使用UTF-8编码，避免乱码问题。代码中不出现中文和拼音，最多注释中可以出现中文。
    类和方法必须有注释wend文档

2.1.1 包的命名规范

  • 一级包名为com，二级包名为xxx（可以是公司域名或者个人命名），三级包名根据应用进 行命名，四级包名为模块名或层级名。如：

    com.automan.activity 
com.automan.adapter

2.1.2 类的命名规范

  • 类的命名规范：类名必须是名词，首字母大写，采用驼峰命名法。类名应该简单清晰。 如果是Android SDK提供的Android应用组成部分，那么类名应该是类的作用的英文全称或者缩写＋ Android SDK基类的名字。

    例子： 

    Activity类名字应该是类似  MainActivity, NaviActivity （Navigation， 导航缩写Navi）, TextActivity, BookActivity
Service类名字缩写应该是类似 DownloadService…

    每个类完成后应该有作者姓名和联系方式的注释，对自己的代码负责。如

    /**
 *User: YedongChen
 * Date: 15-12-13
 * Time: 15：32
 * Email: cyd1058@126.com
 * BaseActivity: 所有activity的基类   该类中抽取一些共有的方法 对activity 进行统一管理
 */  
public class BaseActivity extends Activity{}

    不一定安照这个格式，但是请写上名字和该模块的功能描述。（可通过建立注释模板实现）

    2.1.3 接口命名规范

  • 命名规则与类一样采用大驼峰命名法，多以able或ible结尾。例如：
    interface Runable | interface Accessible

    2.1.4 成员变量命名规范

  • 采用小驼峰命名法。采用有意义的命名（meaningful names）。变量的名字必须准确反映它的含义和内容.

  • 除了临时使用的如循环变量等以外，不要使用单字符的变量名。对于使用单字符的临时变量名，建议循环变量名用i,j,k，n表示整数。

    2.1.5 方法命名规范

    采用小驼峰命名法。如有重载方法 重载方法应该放到一块。

    2.1.6 常量命名规范

    全部大写，单词之间用下划线分隔。如

    public static final int MIN_WIDTH = 4;

    2.1.7 控件命名规范

  • 类中控件名称应该与xml布局id保持一致(可以去掉{module_name})。例如:

    在布局文件中 Button 的id为: android:id="@+id/btn_pay"
private Button btn_pay;

  • 命名模式为：{layoutname}{view缩写}_{view的逻辑名称}，如：

    主界面LinearLayout 的布局id –> mainactivity_ll_content
模块简称为qz的 ImageView 的布局id –>mainactivity_iv_photo

    2.1.8 图片资源文件命名规范

  • 图标命名：{layout_name}ic{名称} 例如：

    activity_home_ic_app.png

  • 背景图片命名： {layout_name}bg{名称} 例如：

    activity_home_navbar_highlight_normal.9.png

  • 按钮Button命名： {layout_name}btn{名称} 例如：

    activity_home_btn_login_normal.9.png

  • 按钮checkbox图片命名：{layout_name}checkbox{名称} 例如：

    activity_home_checkbox_cart_true.png

    2.1.9 布局文件命名规范

  • 全部小写，采用下划线命名法。
    activity layout： activity_{名称} 例如：

    activity_main.xml | activity_shopping.xml

  • fragment layout:fragment_{名称} 例如：

    fragment_main.xml | fragment_shopping.xml

  • Dialog layout: dialog_{名称} 例如：

    dialog_loading.xml

  • 列表项布局命名：{activity_name}_listitem{名称} 例如：

    mainactivity_item_productdetail.xml

  • 包含项布局命名：include_{名称} 例如：

    include_head.xml

  • widget layout： widget_{名称} 例如：

    widget_shopping_detail.xml
  • 控件常用的缩写名称，如下所示：
组件名称         简写            组件名称         简写

Button            Btn            RadioButton        Rbtn

ImageButton        Ibtn        TextView        Tv

ImageView        Iv            ListView        Lv

ProgressBar        Pbar        EditText        Et

ScrollView        Sv            CheckBox        Cb

RelativeLayout    Rly            LinearLayout    Lly

TableLayout        Tly            

FrameLayout        Fly

3 架构设计

  • 遵循 mvc模式
  • 将共有的方法与属性进行抽取 形成父类
  • 遵循低耦合高内聚的原则

4 Android开发注意事项与开发技巧

  • 养成提交代码的习惯，每次提交之间需更新代码，以免造成冲突。对于提交的内容要做注释说明。
  • 尽量在声明局部变量的同时初始化。
  • 对于耗时的操作一定要放在子线程中去执行。如访问网络，数据库操作等.利用时间差 将耗时的操作分时间段执行提高性能。
  • 不在 Activity、Service、BroadcastReceiver 的生命周期等对响应时间敏感函数执行耗时操作，可适当 delay。
  • 在用静态变量的时候 一定要减少对对象的引用。否则容易造成内存泄露。
  • 使用Bitmap及时调用recycle()。虽然调用recycle()并不能保证立即释放占用的内存，但是可以加速Bitmap的内存的释放。
  • activity中在一个View.OnClickListener中处理所有的逻辑 不建议让activity 或fragment 实现接口处理逻辑。
  • 逻辑判断一定要严谨。
  • 界面优化
    • 使用 include merge 标签 重用布局
    • 首次不需要使用的节点设置为 GONE 或使用 viewstub
    • 复杂布局可考虑用RelativeLayout
  • listview的优化
    • 重用convertView
    • 使用ViewHolder
    • 分页加载
    • 通过监听Listview的状态实现滑动时不加载数据，停下来时加载数据
  • OOM异常的处理
    • 建立图片缓存机制　利用LRU(Last Recently Use)算法。可利用第三方开源库实现 比如imageLoader、 fresco等。
      • 用弱引用
      • 大的占内存的对象在不用的时候 回收。
      • 不要出现死循环的操作
      • 减少对象的初始化
      • 自定义控件，我们可以通过canvas.clipRect()来避免 OverDraw
  • 内存优化
    • 减少不必要的网络访问 能一次获取的数据就不多次获取
    • 用IntentService代替Service,用LocalBroadCast发送广播
    • 使用Application的Context。如果想保持一个长期生命的对象，并且这个对象需要一个Context，就可以使用Application对象。可以通过调用Context.getApplicationContext()方法或者Activity.getApplication()方法来获得Application对象。
    • 及时关闭资源

关于性能的优化 具体可参考http://www.trinea.cn/android/performance/
